Curnagl

The Core Interface for High-Performance Slurm Clusters

Overview

The Curnagl Interface is the backbone module designed to interact with massive HPC clusters running Slurm, specifically the Curnagl cluster at UNIL. It handles the complexity of job allocation, node orchestration, and environment synchronization at scale.

Unlike standard sbatch scripts, SlurmRay dynamically deploys a Ray cluster inside your Slurm allocation, giving you a unified Python interface to control hundreds of cores directly from your local IDE.

Usage with RayLauncher

To use the Curnagl backend, simply set cluster="curnagl" (or your custom cluster name) when initializing RayLauncher. 

from slurmray import RayLauncher

# Initialize SlurmRay for Curnagl Cluster
launcher = RayLauncher(
    project_name="genomic_analysis_v4",
    cluster="curnagl",  # Activates Curnagl/Slurm backend
    node_nbr=4,         # Number of nodes
    max_running_time=120, # 2 hours
    memory=64           # 64GB per node
)

Argument Reference

Argument Type Description Default
cluster str Must be set to "curnagl" (or custom IP) to activate the Slurm interface. "curnagl"
node_nbr int Number of separate physical nodes to allocate. 1
use_gpu bool If True, loads CUDA/cuDNN modules and requests GPU resources in Slurm. False
memory int RAM to request per node (in GB). 64
max_running_time int Max duration of the job in minutes. Slurm will kill the job after this time. 60
modules list[str] List of Slurm modules to load (e.g. ["gcc/11.2.0"]). Note: The launcher loads default GCC/Python/CUDA automatically. []

Advanced Features

Automated Module Management

The Curnagl backend keeps your environment sane. It automatically loads the optimal versions of GCC, Python, and CUDA that match the Desi-Labs standards, ensuring binary compatibility for compiled packages.

Job Cleanup

The cancel() method sends a scancel command to the cluster, instantly terminating the job and freeing up your allocation priority.
