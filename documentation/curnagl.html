Curnagl

The Core Interface for High-Performance Slurm Clusters

Overview

‚ö†Ô∏è Stability Warning: The Curnagl backend has not been maintained recently and may not be fully stable. If you encounter any issues, please open an issue on GitHub.

The Curnagl Interface is the backbone module designed to interact with massive HPC clusters running Slurm, specifically the Curnagl cluster at UNIL. It handles the complexity of job allocation, node orchestration, and environment synchronization at scale.

Unlike standard sbatch scripts, SlurmRay dynamically deploys a Ray cluster inside your Slurm allocation, giving you a unified Python interface to control hundreds of cores directly from your local IDE.

Usage with RayLauncher

To use the Curnagl backend, simply set cluster="curnagl" when initializing RayLauncher. 

from slurmray import RayLauncher

# Initialize SlurmRay for Curnagl Cluster
launcher = RayLauncher(
    project_name="genomic_analysis_v4",
    cluster="curnagl",  # Activates Curnagl/Slurm backend
    node_nbr=4,         # Number of nodes
    max_running_time=120, # 2 hours
    memory=64,          # 64GB per node
    num_cpus=16         # 16 CPUs per node
)

Log Locations

If something goes wrong, you can find logs in:

  • Local: .slogs/RayLauncher.log (detailed bridge logs)
  • Remote: Standard Slurm output files (e.g., slurm-XXXX.out) located in your project directory on the cluster.

Argument Reference (Slurm-Specific)

Argument Type Description Default
project_name str Required. Unique project identifier. Consistent naming allows virtual environment reuse. None
cluster str Must be "curnagl" (or your custom cluster IP/hostname). "curnagl"
node_nbr int Number of separate physical nodes to request. 1
num_cpus int Number of CPUs to use per node. 4
memory int RAM per node (GB). 20
num_gpus int Number of GPUs per node. Auto-loads CUDA/CuDNN modules. 0
files list[str] Extra data/config files to sync. (Code is auto-detected). []
modules list[str] Slurm modules to load (e.g. ["gcc/13.2.0"]). []
max_running_time int Max duration in minutes. 60
runtime_env dict Ray runtime environment configuration. {}
retention_days int Days to keep remote files and venvs (1-30). 1
force_reinstall_venv bool Forces a clean re-installation of the virtual environment. False
force_reinstall_project bool Cleans remote project files before upload. False
asynchronous bool Return a tracking object immediately. False

Advanced Orchestration

Automated Module Management

The Curnagl backend keeps your environment sane. It automatically loads the optimal versions of GCC, Python, and CUDA that match the Desi-Labs standards (e.g., GCC 13.2.0, Python 3.12.1), ensuring binary compatibility for compiled packages.

Transparent Dependencies

SlurmRay's AST scanner ensures that all local packages and imported scripts are pushed to the cluster's temporary workspace. No manual setup or rsync commands are required.

Job Cleanup

The cancel() method or the SlurmRay CLI can be used to send a scancel command, instantly terminating the job and freeing up cluster resources.
