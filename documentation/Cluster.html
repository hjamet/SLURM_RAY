RayLauncher API Reference

Complete documentation for the SlurmRay Python interface.

Class: RayLauncher

The main orchestrator class. Initializes a connection to a specific cluster or local environment and manages the lifecycle of the job.

Constructor

__init__(project_name, files=[], modules=[], node_nbr=1, num_gpus=0, memory=20, num_cpus=4, max_running_time=60, runtime_env={}, server_run=True, server_ssh=None, server_username=None, server_password=None, log_file=".slogs/Cluster.log", cluster="curnagl", force_reinstall_venv=False, force_reinstall_project=False, retention_days=1, asynchronous=False)
Auto-Detection: SlurmRay automatically detects Python source files imported by your task. Only use the files parameter for non-Python assets like data or configs.
Parameter Type Default Description
project_name str None Required. Unique identifier for the project. Used for organization and venv reuse.
cluster str 'curnagl' Target environment: 'curnagl', 'desi', 'local', or a custom IP.
files List[str] [] List of extra local files/directories to synchronize (relative to project root).
modules List[str] [] Slurm modules to load (e.g., ['gcc/13.2.0']). Ignored on Desi.
node_nbr int 1 Number of nodes to allocate. Forced to 1 for 'desi'.
num_gpus int 0 Number of GPUs to request per node. Auto-loads CUDA/CuDNN on Slurm.
num_cpus int 4 Number of CPUs to use per node.
memory int 20 RAM to request per node in GB.
max_running_time int 60 Maximum execution time in minutes.
runtime_env dict {'env_vars': {}} Ray runtime environment configuration.
server_run bool True If False, forces local execution.
force_reinstall_venv bool False Wipe and recreate the remote virtual environment.
force_reinstall_project bool False Clean the remote project directory before upload.
retention_days int 7 Days to keep remote files and venvs (1-30).
asynchronous bool False Return a FunctionReturn object immediately.

Methods

__call__

__call__(func, args={}, cancel_old_jobs=True, serialize=True) -> Any

Executes the given function on the configured cluster.

Parameter Type Default Description
func Callable - The function to execute.
args dict {} Keyword arguments for the function.
cancel_old_jobs bool True Cancels previous jobs for this project before starting.

Returns: The result of the function, or a FunctionReturn object if asynchronous=True.

cancel

cancel(target=None)

Cancels a running job.

Parameter Type Default Description
target Any None Job ID or FunctionReturn object. If None, cancels the last started job.

Associated Objects

class FunctionReturn

Returned when asynchronous=True. Allows non-blocking monitoring of the job.

Properties

  • result: Returns the computation result if ready, or "Compute still in progress".
  • logs: Generator yielding log lines.

Methods

  • cancel(): Cancels the job associated with this result.
