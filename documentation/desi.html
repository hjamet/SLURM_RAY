Desi

The Future of Standalone Server Orchestration

Overview

The Desi Interface is specifically designed for standalone servers and internal lab machines (like the powerful Desi-Lab servers). It is optimized for rapid iteration, immediate feedback, and bypassing the queue times often associated with Slurm clusters.

While the Curnagl interface handles the heavy lifting of multi-node clusters, the Desi interface provides a lightweight, agile environment that feels exactly like running code on your own laptopâ€”but with 128 cores and 500GB of RAM.

Complete Example

This example demonstrates a typical workflow using the Desi backend to train a model on a remote server. Notice how minimal the change is from local code. 

from slurmray import RayLauncher
import time

# 1. Define the task locally
def train_and_evaluate(seed, epochs):
    print(f"Starting training with seed {seed}")
    # Simulate heavy work
    time.sleep(10) 
    accuracy = 0.95 + (seed * 0.001)
    return {"seed": seed, "accuracy": accuracy}

# 2. Configure for Desi Server
launcher = RayLauncher(
    project_name="desi_demo_v1",
    cluster="desi",        # Activates Desi Interface
    server_username="henri", # Optional if set in .env
    files=["data/"]       # Upload local data folder
)

# 3. Execute Remote Task
print("ðŸš€ Launching task to Desi...")
result = launcher(
    train_and_evaluate, 
    args={
        "seed": 42, 
        "epochs": 100
    }
)

print(f"âœ… Received Result: {result}")

Detailed Argument Reference

Argument Type Required Description
cluster str Yes Set to "desi" to activate the Desi interface.
project_name str Yes Unique identifier for the workspace. Creates ~/.slogs/project_name on the remote server.
files list[str] No List of local files/dirs to sync. Auto-sync via rsync.
server_username str No SSH username. Defaults to DESI_USERNAME in .env.
force_reinstall_venv bool No If True, wipes and reinstalls the remote venv.
runtime_env dict No Ray runtime environment (e.g., environment variables).
retention_days int No Days to keep files on server (default: 7).
node_nbr int No Ignored (always 1).
