Desi

The Future of Standalone Server Orchestration

Overview

The Desi Interface is specifically designed for the private standalone server of the department (ISIPOL09). It is a powerful machine equipped with:

  • 2x NVIDIA GeForce RTX 3090 (24 GB VRAM each)
  • 125 GB of RAM
  • 24 CPUs
Using cluster="desi" is equivalent to pointing to the IP "130.223.73.209".

üåê Network Requirement: You must be connected to the UNIL network (on-campus or via VPN) to access the Desi server.
No Manual Source Push: SlurmRay automatically scans your local imports and pushes necessary code. You only need to specify data files or non-Python resources in the files argument.

Authentication

To use Desi, you must provide your credentials. The recommended way is to add them to your .env file at the project root:

DESI_USERNAME=your_username
DESI_PASSWORD=your_password

Alternatively, you can set them as environment variables.

Examples

1. Standard Synchronous Execution

Notice how we specify the required resources. Please be precise to optimize the scheduling for other users. 

from slurmray import RayLauncher
from src.my_model import train

# Configure for Desi Server
launcher = RayLauncher(
    project_name="alpha_model_v1",
    cluster="desi",
    num_cpus=8,             # Request 8 cores
    memory=32,               # Request 32GB RAM
    num_gpus=1,              # Request 1 GPU
    files=["data/config.json"]
)

# Blocks until the result is returned
result = launcher(train, args={"lr": 0.001})
print(f"‚úÖ Result: {result}")

2. Asynchronous Monitoring

The asynchronous mode returns a tracking object immediately. You can check the result property, which will return "Compute still in progress" until the task is finished. 

from slurmray import RayLauncher
import time

launcher = RayLauncher(project_name="async_test", cluster="desi", asynchronous=True)

# Returns a FunctionReturn object immediately
job = launcher(heavy_task)

# Monitor result
while job.result == "Compute still in progress":
    print("Still waiting...")
    time.sleep(10)

print(f"‚úÖ Done: {job.result}")

Best Practices for Desi

1. Intelligent Project Naming

Use the same project_name for all calculations belonging to the same project. SlurmRay checks the hash of your requirements: if they haven't changed, the remote virtual environment is reused instantly, avoiding redundant installations.

2. Automatic Dependency Detection

While SlurmRay is smart, it has limits. Static imports are detected perfectly, but dynamic imports (e.g., importlib.import_module) or files accessed via dynamic paths in open() might be missed. If you encounter "ModuleNotFound" errors on the server, add the missing items to the files list manually.

3. Resource Monitoring with CLI

Always use the SlurmRay CLI to monitor your jobs: 

slurmray desi
This allows you to see real-time job status, cancel stalled tasks, and open the Ray Dashboard via an automated SSH tunnel.

4. Automatic Cleanup

To prevent server congestion, all project files and virtual environments are automatically deleted after the specified retention_days (default is 1 day).

Log Locations

If something goes wrong, you can find logs in:

  • Local: .slogs/RayLauncher.log (detailed bridge logs)
  • Remote: ~/slurmray-server/{project_name}/.slogs/server/ (standard output of your task)
