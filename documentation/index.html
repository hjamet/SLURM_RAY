Quick Start

Transform a local script into a distributed task in seconds:

from slurmray import RayLauncher

# 1. Define your logic
def compute_heavy_pi(n):
    # This code runs on the remote cluster nodes!
    return 3.1415926535

# 2. Configure the bridge
launcher = RayLauncher(
    project_name="pi_calc",
    cluster="curnagl",
    node_nbr=4,
    num_gpus=1
)

# 3. Execute!
result = launcher(compute_heavy_pi, args={"n": 10**8})

API Reference

class Cluster

The main entry point for orchestrating jobs across local and remote environments.

Argument Type Default Description
project_name str None Required. Name of the project integration.
cluster str 'curnagl' Target environment: 'curnagl', 'desi', 'local', or custom IP.
files List[str] [] List of relative file paths to synchronize.
modules List[str] [] Slurm modules to load (e.g. ['gcc/13.2.0']). Ignored on Desi.
node_nbr int 1 Number of nodes. Fixed to 1 for Desi/Local.
num_gpus int 0 Number of GPUs to request. Auto-loads CUDA/CuDNN if > 0.
memory int 20 RAM per node in GB.
num_cpus int 4 Number of CPUs to use per node.
max_running_time int 60 Wall time limit in minutes.
runtime_env dict {'env_vars': {}} Ray runtime environment configuration.
server_run bool True Enable remote execution. If False, runs locally.
server_ssh str None SSH host. Auto-detected from cluster config.
server_username str None SSH user. Auto-loaded from .env (CURNAGL_USERNAME etc.).
server_password str None SSH password. Auto-loaded from .env.
log_file str '.slogs/Cluster.log' Path to log file.
force_reinstall_venv bool False Wipe and recreate the remote virtualenv.
retention_days int 7 Days to keep remote files/venv (1-30).
force_reinstall_project bool False Force project files cleanup before upload.
asynchronous bool False Return immediately with a tracking object.
